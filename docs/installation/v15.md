## 1. Step-by-Step Server & ERPNext Setup

> ⚠️ **IMPORTANT:** Never run `sudo pip install` — it corrupts the system Python (3.12). Always use isolated virtual environments.

---

### 1.1 System Prerequisites

Update the system and install core dependencies before touching Python or Bench.

```bash
# 1. Change hostname if required
## check current host name
hostname
hostnamectl
## update host name
sudo hostnamectl set-hostname <new-hostname>
## verify change
hostname
sudo reboot

# 2. Change timezone if required
## check current system timezone
sudo timedatectl
## get list of available timezone
timedatectl list-timezones | grep America
## update timezone
sudo timedatectl set-timezone America/New_York

# 3. Update & upgrade all packages
sudo apt update && sudo apt upgrade -y

# 4. Install essential packages
sudo apt install -y git curl wget unzip build-essential \
    software-properties-common \
    libssl-dev libffi-dev libjpeg-dev libpng-dev \
    libxrender1 fonts-liberation \
    nginx redis-server supervisor pkg-config -y

# 5. Install wkhtmltopdf (required for PDF print in ERPNext)
sudo apt install xvfb libfontconfig -y
sudo apt install -y fontconfig xfonts-75dpi xfonts-base
wget "https://github.com/wkhtmltopdf/packaging/releases/download/0.12.6.1-2/wkhtmltox_0.12.6.1-2.jammy_amd64.deb"
sudo dpkg -i wkhtmltox_file.deb
sudo apt --fix-broken install -y
wkhtmltopdf --version

# 6. Setup Bitbucket/Github SSH Key (for custom app repo)


# 7. Install Snapd
sudo apt install snapd -y
sudo systemctl enable snapd
sudo systemctl start snapd
sudo snap install core
sudo snap refresh core

# 8. Install Certbot
sudo snap install --classic certbot
sudo ln -sf /snap/bin/certbot /usr/bin/certbot
## verify version
certbot --version

```

### 1.2 Install Python 3.10 (ERPNext Runtime)

> ℹ️ Python 3.12 ships with Ubuntu 24.04 and must **NOT** be touched. ERPNext v15 runs on Python 3.10 in an isolated venv.

```bash
# Add deadsnakes PPA for Python 3.10
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update

# Install Python 3.10 and companions
sudo apt install -y python3.10 python3.10-venv python3.10-dev python3-setuptools python3-pkg-resources
python3.10 -m ensurepip --upgrade
python3.10 -m pip install --upgrade pip

# Verify installation
python3.10 --version   # Expected: Python 3.10.x
```

### 1.3 Install & Configure MariaDB

```bash
# Install MariaDB 10.6
sudo apt install -y mariadb-server mariadb-client

# Secure installation
sudo mysql_secure_installation
# Answer:
#   Set root password        -> Y
#   Remove anonymous users   -> Y
#   Disallow root remote login -> Y
#   Remove test database     -> Y
#   Reload privilege tables  -> Y

# Enable & start MariaDB
sudo systemctl enable mariadb
sudo systemctl start mariadb
sudo systemctl status mariadb
```

**Create dedicated ERPNext DB user:**

```sql
sudo mysql -u root -p

-- Inside MySQL shell:
CREATE USER 'frappe'@'localhost' IDENTIFIED BY '<password>';
CREATE USER 'frappe'@'127.0.0.1' IDENTIFIED BY '<password>!';
GRANT ALL PRIVILEGES ON *.* TO 'frappe'@'localhost' WITH GRANT OPTION;
GRANT ALL PRIVILEGES ON *.* TO 'frappe'@'127.0.0.1' WITH GRANT OPTION;
FLUSH PRIVILEGES;
EXIT;
```

### 1.5 Setup Bench Environment

> ℹ️ Install Bench in `/opt/bench-env` so it is accessible via `sudo` (needed for production setup).

```bash
# Install frappe-bench inside it
sudo -H python3.10 -m pip install frappe-bench
chmod -R o+rx /home/$USER/

# Symlink bench to system PATH
sudo ln -sf /opt/bench-env/bin/bench /usr/local/bin/bench

# Verify
bench --version
sudo which bench # show /usr/local/bin/bench
```

### 1.6 Initialise Bench & Install ERPNext

```bash
# Navigate to home (or preferred install directory)
cd /home/$USER

# Initialise bench with Python 3.10
bench init frappe-bench --python python3.10 --frappe-branch version-15

cd frappe-bench

# Setup Multitenancy
bench config dns_multitenant on

# Create a new site
bench new-site yoursite.com --db-name <database_name>
bench use yoursite.com
# Enter MySQL root password when prompted
# Set Administrator password for the site

# Get ERPNext app
bench get-app erpnext --branch version-15

# Install ERPNext on the site
bench --site yoursite.com install-app erpnext

# List installed app
bench list-apps
```

## 2. MariaDB Custom Config for Performance Tuning

> ℹ️ Adjust values based on your server RAM. Values below are calibrated for a **4 GB RAM** VPS. Multiply proportionally for higher RAM.

---

### 2.1 Create Performance Config File

Create a dedicated override file so system upgrades don't wipe your tuning:

```bash
sudo nano /etc/mysql/mariadb.conf.d/99-erpnext-tuning.cnf
```

---

### 2.2 Full MariaDB Performance Config

Paste the following into `99-erpnext-tuning.cnf`:

```ini
[mysqld]

# ── Character Set ─────────────────────────────────────────
character-set-client-handshake = FALSE
character-set-server           = utf8mb4
collation-server               = utf8mb4_unicode_ci

# ── InnoDB Buffer Pool ────────────────────────────────────
# Set to 50-70% of available RAM
innodb_buffer_pool_size        = 2G
innodb_buffer_pool_instances   = 2
innodb_buffer_pool_chunk_size  = 128M

# ── InnoDB I/O Performance ───────────────────────────────
innodb_file_per_table          = 1
innodb_flush_log_at_trx_commit = 2        # Slightly relaxed durability for speed
innodb_flush_method            = O_DIRECT # Bypass OS cache
innodb_log_file_size           = 512M
innodb_log_buffer_size         = 64M
innodb_io_capacity             = 400
innodb_io_capacity_max         = 800
innodb_read_io_threads         = 8
innodb_write_io_threads        = 8

# ── Connection Settings ───────────────────────────────────
max_connections                = 300
max_connect_errors             = 10000
wait_timeout                   = 300
interactive_timeout            = 300
connect_timeout                = 10

# ── Query Cache (disabled for MariaDB 10.6+) ─────────────
query_cache_type               = 0
query_cache_size               = 0

# ── Thread & Sort Buffers ────────────────────────────────
thread_cache_size              = 100
sort_buffer_size               = 4M
join_buffer_size               = 4M
read_buffer_size               = 2M
read_rnd_buffer_size           = 4M
tmp_table_size                 = 128M
max_heap_table_size            = 128M

# ── Slow Query Logging ───────────────────────────────────
slow_query_log                 = 1
slow_query_log_file            = /var/log/mysql/slow-queries.log
long_query_time                = 2
log_queries_not_using_indexes  = 0

# ── General & Binary Logging ─────────────────────────────
general_log                    = 0   # Keep OFF in production
skip-log-bin

# ── Table & Open File Limits ─────────────────────────────
table_open_cache               = 4000
table_definition_cache         = 2000
open_files_limit               = 65535

# ── Networking ──────────────────────────────────────────
bind-address                   = 127.0.0.1
max_allowed_packet             = 256M

[client]
default-character-set          = utf8mb4

[mysql]
default-character-set          = utf8mb4
```

## 3. All Issue Fixes & Resolution Commands

### 3.0 Issues Summary Table

| # | Issue | Root Cause | Fix |
|---|-------|------------|-----|
| 4.1 | Python `_distutils_hack` error | `sudo pip` corrupted system Python 3.12 | Remove broken dist-packages; reinstall python3.12 packages |
| 4.2 | `supervisorctl pkg_resources` error | `setuptools` missing in system Python | Reinstall `python3-setuptools` & `python3-pkg-resources` |
| 4.3 | `fail2ban` not starting | Python env corruption breaks fail2ban | Reinstall fail2ban after fixing Python env |
| 4.4 | `bench: command not found` (sudo) | bench installed in user PATH only | Install in `/opt/bench-env`; symlink to `/usr/local/bin` |
| 4.5 | MariaDB Access Denied for ERPNext | User missing for `127.0.0.1` host | Create `user@127.0.0.1` with full privileges |
| 4.6 | Supervisor `no such process` | Wrong process name in supervisorctl | Use `frappe-bench-*` pattern, not `frappe` |
| 4.7 | Certbot Python / snapd error | Broken system Python affects certbot | Install Certbot via Snap (not apt) |
| 4.8 | `cp` command errors with `*.*` pattern | Shell alias or glob expansion issues | Replace `cp` with `rsync` for safe file copy |

---

### 3.1 Fix: Python `_distutils_hack` Error

> **Root cause:** Running `sudo pip install` installed packages into system Python 3.12's dist-packages, breaking the `_distutils_hack` module used by setuptools.

```bash
# Step 1: Remove conflicting packages from system Python
sudo rm -rf /usr/local/lib/python3.12/dist-packages/*

# Step 2: Reinstall python3.12 base packages from apt
sudo apt install --reinstall python3.12 python3.12-distutils

# Step 3: Fix setuptools
sudo apt install --reinstall python3-setuptools python3-pip

# Step 4: Verify Python 3.12 is healthy
python3 -c "import setuptools; print(setuptools.__version__)"
python3 -c "import distutils; print('OK')"
```

---

### 3.2 Fix: `supervisorctl pkg_resources` Error

> **Root cause:** The supervisor process calls python3 at startup; if setuptools/pkg_resources is missing or broken, supervisorctl fails immediately.

```bash
# Reinstall setuptools and pkg-resources for system Python
sudo apt install --reinstall python3-setuptools python3-pkg-resources

# Reinstall supervisor itself
sudo apt install --reinstall supervisor

# Reload systemd and restart supervisor
sudo systemctl daemon-reload
sudo systemctl restart supervisor

# Verify supervisor is running
sudo systemctl status supervisor
sudo supervisorctl status
```

---

### 3.3 Fix: `fail2ban` Not Starting

> **Root cause:** fail2ban depends on the system Python. After Python corruption it fails silently or raises ImportError on startup.

```bash
# First ensure Python 3.12 is healthy (see fix 3.1)

# Reinstall fail2ban
sudo apt remove --purge fail2ban -y
sudo apt install fail2ban -y

# Enable and start
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# Verify
sudo fail2ban-client status
```

---

### 3.4 Fix: `bench` Command Not Found (sudo)

> **Root cause:** When bench is installed via pip as a regular user, it goes into `~/.local/bin` which is NOT in sudo's secure PATH.

```bash
# Correct approach: install bench in a system-wide venv
sudo python3.10 -m venv /opt/bench-env
sudo /opt/bench-env/bin/pip install --upgrade pip
sudo /opt/bench-env/bin/pip install frappe-bench

# Create system-wide symlink
sudo ln -sf /opt/bench-env/bin/bench /usr/local/bin/bench

# Verify both user and sudo can see it
bench --version
sudo bench --version

# If old user-local bench exists, remove it
rm -f ~/.local/bin/bench
```

---

### 3.5 Fix: MariaDB Access Denied for ERPNext

> **Root cause:** ERPNext connects via both `localhost` (Unix socket) and `127.0.0.1` (TCP). If only the `localhost` user exists, TCP connections are denied.

```sql
sudo mysql -u root -p

-- Check existing users
SELECT User, Host FROM mysql.user WHERE User='frappe';

-- Create the missing TCP user
CREATE USER IF NOT EXISTS 'frappe'@'127.0.0.1' IDENTIFIED BY 'YourPassword';
GRANT ALL PRIVILEGES ON *.* TO 'frappe'@'127.0.0.1' WITH GRANT OPTION;

-- Also ensure localhost user exists
CREATE USER IF NOT EXISTS 'frappe'@'localhost' IDENTIFIED BY 'YourPassword';
GRANT ALL PRIVILEGES ON *.* TO 'frappe'@'localhost' WITH GRANT OPTION;

FLUSH PRIVILEGES;
EXIT;
```

---

### 3.6 Fix: Supervisor `no such process`

> **Root cause:** Supervisor process names are auto-generated from the bench directory name. The default `frappe` is wrong; the correct pattern is `frappe-bench-*`.

```bash
# List actual process names
sudo supervisorctl status

# Example correct commands:
sudo supervisorctl restart frappe-bench-web:frappe-bench-frappe-web
sudo supervisorctl restart 'frappe-bench-workers:*'
sudo supervisorctl restart 'frappe-bench-web:*'

# Restart ALL bench processes at once
sudo supervisorctl restart 'frappe-bench-*:*'

# Reread config if you edited supervisor conf
sudo supervisorctl reread
sudo supervisorctl update
```

---

### 3.7 Fix: Certbot Python / Snapd Error

> **Root cause:** The apt-installed certbot uses system Python. With a broken Python environment, certbot fails. The Snap version is fully self-contained.

```bash
# Remove apt-installed certbot if present
sudo apt remove certbot -y

# Install via Snap (self-contained, no system Python dependency)
sudo snap install --classic certbot

# Symlink to PATH
sudo ln -sf /snap/bin/certbot /usr/bin/certbot

# Verify
certbot --version

# Obtain certificate via bench
sudo bench setup lets-encrypt yourdomain.com

# Or manually with certbot
sudo certbot --nginx -d yourdomain.com
```

---

### 3.8 Fix: `cp` Command Errors (`*.*` Pattern)

> **Root cause:** Shell aliases or bash globbing can cause `cp -r *.* /dest` to fail when filenames contain multiple dots or alias overrides are present.

```bash
# Always use rsync instead of cp for safe file copying

# Basic rsync (preserves permissions)
rsync -av source/ destination/

# rsync with specific pattern
rsync -av --include='*.conf' --exclude='*' source/ dest/

# If cp is aliased, bypass the alias
\cp -r source/ destination/   # backslash bypasses alias

# Or explicitly use full path
/usr/bin/cp -r source/ destination/
```

## 4. Security Hardening

### 4.1 UFW Firewall

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
sudo ufw status verbose
```

---

### 4.2 SSH Key Auth & Hardening

```bash
# On LOCAL machine: generate key pair
ssh-keygen -t ed25519 -C 'your@email.com'
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@your-server-ip

# On SERVER: harden SSH config
sudo nano /etc/ssh/sshd_config

# Set these values:
#   PermitRootLogin no
#   PasswordAuthentication no
#   PubkeyAuthentication yes
#   MaxAuthTries 3
#   LoginGraceTime 30

sudo systemctl restart sshd
```

---

### 4.3 Fail2Ban Configuration

```bash
# Create local jail config
sudo nano /etc/fail2ban/jail.local
```

```ini
[DEFAULT]
bantime  = 3600
findtime = 600
maxretry = 5

[sshd]
enabled  = true
port     = 22
logpath  = %(sshd_log)s

[nginx-http-auth]
enabled  = true
```

```bash
# Apply config
sudo systemctl restart fail2ban
sudo fail2ban-client status
```
